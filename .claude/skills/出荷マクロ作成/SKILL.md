# 出荷マクロ作成スキル

ネットショップの「出荷データ作成」をExcel VBAで自動化するためのスキル。
株式会社SEEDのfuuブランドが、複数モール（Amazon／楽天／Wowma／ペライチ）で
同じ作業フローを使っているため、**1つの型を覚えれば全チャネルに展開できる**。

このスキルの目的は、新しいチャネルの追加や既存マクロの修正を頼まれたとき、
毎回ゼロから設計せず、**実証済みの同じ構造・同じ語彙・同じ進め方**で仕上げること。

## 大原則（会社の憲法に従う）

- **嘘をつかない・隠蔽しない・確認する**。動くか確かめていない部分は「未確認」と正直に言う。
- **7割でOK、残り3割は人が仕上げて次へ改善**。完璧を待たず、小さく試して回す。
- **発注・請求のような機密度の高い処理ではないが、出荷データは実運用に直結する**。
  作ったマクロは必ず1件でテストし、最終的に山本社長が内容を確認する前提で渡す。
- **作ったら即push**（同期ルール）。

---

## 五郎の型（4チャネル共通の構造）

どのチャネルのマクロも、**3つのボタンで全工程を制御する**。これが型の中心。

```
実行1_<チャネル>_編集データ作成
    → 元データ → 「編集」シートへ転記 + 発送方法を自動判定してA列に書き込む
    → 人がA列を目視確認・修正できるよう一度止まる（送料の安い方へ手動変更するため）

実行2_<チャネル>_出荷データ作成
    → 編集シートを発送方法ごとに各「発行用_編集から貼付」シートへ自動振り分け
    → 物出しリスト作成、クリックポスト/佐川の発行用アップデータ作成

実行3_<チャネル>_通知CSV作成
    → 追跡番号レポートを貼り付けた後、通知用データを集計しCSV出力
```

各「実行」は内部で `手順1〜8` の個別マクロを `Call` で順番に呼ぶだけにする。
こうすると、既存の手作業マクロ（手順N）を壊さずにボタン化でき、
不具合時もどの手順で止まったか切り分けやすい。

### 各実行ボタンのお約束

- `Application.ScreenUpdating = False / True` で囲む（処理中のちらつき防止＆高速化）。
- `On Error GoTo ErrHandler` でエラーを拾い、`MsgBox` で「どの手順で何が起きたか」を出す。
- 処理の最後に **次にやること（次のステップ）を `MsgBox` で案内する**。
  人が次の操作（ラベル発行、追跡番号貼付など）を迷わないようにするのが狙い。

## 発送方法の語彙（絶対に統一する）

「編集」シートA列に入る発送方法は、**この6種類で固定**。表記ゆれは厳禁。
振り分け・フィルタ・物出しが全部この文字列で動いているため、1文字でも変えると壊れる。

```
ペットクリック1通 / ペットクリック複数
衣類クリック1通   / 衣類クリック複数
ペット佐川        / 衣類佐川
```

判定の考え方（チャネル共通）:
1. 商品名に衣類キーワードが含まれるか → `Is衣類<チャネル>()` で判定（ペット or 衣類）。
2. 同一送付先（氏名＋住所 や 姓名＋都道府県＋市区）の件数を `COUNTIFS` で数える
   → 2件以上なら「複数」、1件なら「1通」。
3. **佐川への変更は基本ここでは自動化しすぎず、人の手動修正に委ねる**。
   理由：3kg等の重い商品は佐川770円よりクリックポスト185円×複数の方が安い場合があり、
   コスト最適化は人が最終判断したいから。実行1の後に一度止めるのはこのため。

## 衣類判定ヘルパー（標準化のポイント）

商品がランジェリー等の「衣類」かどうかを商品名で判定する関数。
**これが現状チャネル間で最もバラついている。新規・修正時はここを揃える。**

推奨する標準形（関数名はチャネル名を付けて衝突を避ける）:

```vba
Function Is衣類<チャネル>(itemName As String) As Boolean
    Dim keywords As Variant
    keywords = Array("脇高", "セクシー", "レース", "リボン", "ブラジャー", "ブラ", _
                     "インナー", "ショーツ", "下着", "ランジェリー", "パンティ", "ビキニ")
    Dim i As Integer
    For i = 0 To UBound(keywords)
        If InStr(itemName, keywords(i)) > 0 Then
            Is衣類<チャネル> = True
            Exit Function
        End If
    Next i
    Is衣類<チャネル> = False
End Function
```

既知の不統一（直すと品質が上がる点）:
- **関数名**：Amazon/Wowma/ペライチは `Is衣類<チャネル>`、楽天だけ `IsClothing`。
  → 新規は `Is衣類<チャネル>` に寄せる。楽天を触るときも改名を検討。
- **キーワード一覧**：楽天は5語のみ（しかも「セクシーレースリボン」を1語に連結）で、
  「ショーツ」「下着」等を取りこぼす恐れ。→ 上記12語の標準リストに揃えるのが安全。
- キーワードは商品が増えたら足すだけでよい設計にし、コメントでその旨を明記する。

## チャネルごとに必ず変わるもの

型は同じでも、**列の位置・シート名・出力先・データ列数はチャネルごとに違う**。
これらは思い込みで書かず、対象の `.xlsm` の実物を見て確認すること。
代表的なマッピングは [references/チャネル別_仕様.md](references/チャネル別_仕様.md) にまとめてある。

確認すべき項目:
- 元データの何列目が「商品名／注文番号／送付先氏名／住所」か
- 最終行を判定する基準列（注文番号の列が安全）
- 「編集から貼付」「アップデータ」「物出し」「通知用データ」など各シートの正式名称
- CSV出力先フォルダのパスとファイル名規則（例：Dropbox配下、日付スタンプ付き）

## 新しいチャネルを追加する手順

1. **お手本を選ぶ**：構造が一番近い既存チャネルを選ぶ
   （クリック＋佐川の標準フローなら楽天かAmazonが完成度高め）。
2. **対象の.xlsmの中身を確認**：元データの列構成、シート名を実際に開いて把握する。
   既存マクロが入っているなら下記「VBA抽出」で取り出して差分を見る。
3. **マスターを書く**：`<チャネル>_Module_マスター.bas` に実行1/2/3＋手順N＋振分を作る。
   列位置・シート名・出力先だけを差し替え、構造と語彙は型のまま。
4. **衣類判定を書く**：`<チャネル>_Module_衣類判定.bas` に標準形の `Is衣類<チャネル>` を置く。
5. **1件でテスト**：実データ1〜数件で実行1→2→3を通し、CSVが想定どおり出るか確認。
6. **正直に報告**：動いた部分／未確認の部分／人が仕上げる3割を明記して山本社長へ。
7. **保存とpush**：`.bas` をこのフォルダに保存し、CLAUDE.mdのプロジェクト構成へ追記して即push。

## 既存の.xlsmからVBAを取り出す（救出）

実物の最新ロジックは多くの場合 Dropbox や「各種フォーマット」フォルダの `.xlsm` の中にある。
リポジトリの `.bas` が古い・無い場合は、**現物から取り出すのが正**。手順は
[references/VBA抽出手順.md](references/VBA抽出手順.md) を参照（oletools/olevbaを使う）。

ポイント:
- 同名で複数版があるときは**更新日時と中身の行数で最新を特定**する
  （新しい手順モジュールが増えていたら、それが本物の最新）。
- 取り出したら、標準モジュール（Module系）の中身を上記の型に整理してから保存する。
  クラスモジュール（Sheet*.cls/ThisWorkbook.cls）は中身が空ならスキップしてよい。

## 実物xlsmを安全に編集する（ボタン・マクロを壊さない）

実行1/2/3ボタン付きの実物 .xlsm のセルを書き換える／ひな型化する時の鉄則。
（2026-07-01、ペライチのマスターひな型化で実証）

- **openpyxl で直接書き換えない。** フォームコントロール（ボタン）や描画が壊れる恐れがある。
  openpyxl は「読み取り・検証」専用にする。
- 編集は **Excel の COM 自動操作（PowerShell）** で行う（マクロ・ボタン・書式を保持）。
- **手順：**
  1. 編集前に必ず原本をバックアップ（同フォルダの `_backup\` 等へ Copy-Item）。元に戻せる状態を作る。
  2. Excel COM を開く時は `$excel.AutomationSecurity = 3`（マクロ強制無効）＋ `$excel.EnableEvents=$false`
     ＋ `$excel.DisplayAlerts=$false`。開いた瞬間にマクロが走らないようにする。
  3. セル編集は `$ws.Cells.Item(行,列).Value2`。データ行の一括消去は `$ws.Rows("2:$last").ClearContents()`
     （1行目の見出しは残す）。納品書の空化は `ペライチ_納品書作成` の clear ロジックと同じセルを空にする。
  4. `$wb.Save()` → `$wb.Close($true)` → `$excel.Quit()` → `ReleaseComObject` ＋ `[GC]::Collect()`
     で Excel.exe の残留を防ぐ。
  5. 保存後に openpyxl＋zipfile で検証：`vbaProject.bin`（マクロ）・`drawing`（ボタン）・`ctrlProp`
     （フォームコントロール）が残っているか、シート総数が変わっていないか、顧客データが0件かを確認。
- **顧客データの検証は値を転写しない。** PII読み出しはブロックされる。「空か／接頭辞で始まるか」等の
  True/False だけを出力してチェックする。
- ひな型化＝顧客データ全消去は **ルール5（顧客データの取り扱い）** に従う（`五郎_共通ルール.md`）。
