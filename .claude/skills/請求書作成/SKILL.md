# 請求書作成スキル

株式会社SEEDの **shop fuu（ペットショップ事業）の卸先請求** を自動化するためのスキル。
CLAUDE.mdで「最初の1体（いま動かすサブエージェント）」に指定されている業務そのもの。

**入力（注文）→ 処理（納品書へ転記・自動集計）→ 出力（取引先ごとの請求書）** の流れを、
毎回ゼロから考えず、実物フォーマットの構造・語彙・進め方をそろえて仕上げる。

## 大原則（会社の憲法に従う）

- **請求書＝お金に直結する機密度の高い仕事**。CLAUDE.mdの承認ルールにより、
  **最後は必ず山本社長が内容と金額を確認・承認**する。五郎は「承認前の下ごしらえ」まで。
- **嘘をつかない・隠蔽しない・確認する**。マッチできなかった商品・数量が読めない注文は
  勝手に埋めず、**必ず一覧で正直に報告**して人に確認を仰ぐ。
- **元の整形済みワークブックを壊さない**。印刷レイアウト・画像・数式が命なので、
  原本は直接上書きせず**必ずバックアップ**を取り、コピーに対して作業する（後述の安全ルール）。
- **7割でOK、残り3割は人が仕上げて次へ改善**。全自動を狙わず、転記＋検証まで固めて渡す。
- **作ったら即push**（同期ルール）。成果物はリポジトリ内の `五郎_成果物/` へ。

---

## いちばん大事な事実（このフォーマットの構造）

実物 `SEED様_納品書フォーマット_1.5【据置】.xlsx` を解析した結果。**1ワークブック＝1取引先＝1か月**。

- **納品書1〜10** … 同じ取引先の **日付別の納品**（例：6/2, 6/10…）。1シート＝1回の納品。
- **請求書** … 納品書1〜10を **自動集計** して月次請求にまとめるシート。
- **データリスト** … 商品マスタ。ここに無い商品名は単価が出ない（＝マッチが全て）。
- **取引先リスト** … 卸先名の一覧（表記ゆれの正解はここ）。

### 人が入力するのは「4つだけ」、あとは数式が自動

納品書シートで**手入力するセルは限られている**（それ以外はVLOOKUP等で自動反映）。

| 入力する場所 | 内容 | 例 |
|---|---|---|
| `A3` | 取引先名（様は`H3`に固定であり不要） | `うさぎ専門店RabbitRuru` |
| `M2` | 発行日（＝その納品の日付） | `2026/7/1` |
| `C12`以降 | **商品名（データリストA列と完全一致）** | `fuu 小動物用　陶器のチモシーポッド　スイーツ　ピンク　` |
| `I12`以降 | 発注番号 | `20260701－1`（日付＋連番、全角ハイフン） |
| `K12`以降 | 数量 | `4` |

自動反映される列（**絶対に手で触らない**）:
- `B`＝卸価格№、`H`＝JAN、`J`＝単価 … `データリスト`からVLOOKUP
- `L`＝金額（単価×数量）、`A`＝行番号（SUBTOTAL）
- 請求書シートの合計（`ご請求合計金額`）も納品書から自動集計

> ⚠️ **C列の商品名は「データリスト」A列と1文字でも違うと単価が空欄になる**。
> A列は `商品名　カラー　サイズ`（全角スペース区切り、末尾にも全角スペースが付くことがある）。
> ここのマッチ精度がこのスキルの品質のすべて。→ `scripts/match_products.py` で機械照合する。

詳細なセル対応・数式は [references/フォーマット構造.md](references/フォーマット構造.md) を必ず参照。

## 取引先ごとの運用（Dropboxの実体）

- 卸先ごとにフォルダ：`Dropbox/ペット事業/販売/【直販卸し先】/<取引先名>/`
- その中に **月次ファイル**：`<取引先>様　202X年X月分　納品書・請求書.xlsx`
- 確定済みは `請求済み/`・`請求確定/` サブフォルダへ移動
- **価格版が3種類**あり、取引先ごとに使い分ける（マスタは
  `Dropbox/ペット事業/販売/納品書・請求書フォーマット/` と `【直販卸し先】/` 直下）：
  - `【据置】＝価格`（2025.9.01〜 据置価格）
  - `【10％アップ】▲10％`
  - `【シャンパン家等】/▼10％` 等の特別価格
  - → **どの版を使うかは取引先ごとに決まっている**。勝手に変えない。既存の月次ファイルが
    使っている版を踏襲する（迷ったら社長に確認）。

## 進め方（注文 → 請求書までの標準フロー）

### 手順1. 注文を読み取る（FAX/メール/LINE → 商品名＋数量＋日付）
- 注文は**手入力で届く前提**（FAX画像・メール本文・LINE）。まず人が読める形にする。
- 抽出するのは **納品日・商品名（できるだけ原文のまま）・数量** の3点。発注番号は日付＋連番で振る。
- 画像/PDFで文字が怪しい所は**推測で埋めない**。「読めない」と印を付けて後で確認。

### 手順2. 商品名を「データリスト」の正式名称にマッチさせる（最重要）
- `scripts/match_products.py` に、対象の月次ファイル（または据置フォーマット）と
  注文明細（商品名・数量）を渡すと、**データリストA列の正式名称へ機械照合**する。
- 出力：`正式商品名 / 卸価格№ / 単価 / 数量 / 金額` の**貼り付け用ブロック** ＋ **未マッチ一覧**。
- **未マッチ・あいまい一致は必ず人へ確認**（別カラー・サイズ違い・廃番の可能性）。ここを飛ばさない。

### 手順3. 納品書シートへ転記する
- 対象取引先の月次ファイルを開き、**その納品日のシート（納品書N）** の
  `A3`(取引先)・`M2`(発行日)・`C12〜`(正式商品名)・`I12〜`(発注番号)・`K12〜`(数量) を入力。
- 自動反映列（B/H/J/L）は触らない。**単価が空欄の行が無いか**を必ず目視（＝マッチ漏れの検出）。
- スクリプトでファイルに直接書き込む場合は、下の「安全ルール」を厳守。

### 手順4. 請求書シートで集計を確認する
- `請求書`シートの `ご請求合計金額` と、各納品書の金額合計が一致するか確認。
- 請求月（`B11`）・取引先名（`A3`は納品書1から自動）が正しいか確認。

### 手順5. 出力して社長へ（承認前まで）
- 取引先ごとに請求書（必要なら納品書も）をPDF化 or 印刷用に整える。
- **動いた部分／未マッチで人に確認が必要な部分／金額の根拠**を添えて山本社長へ渡す。
- 承認後、月次ファイルを `請求済み/` へ。成果物は `五郎_成果物/請求書/` にも保存して即push。

## 安全ルール（機密作業ゆえ厳守）

ワークブックには印刷設定・画像・多数の数式・配列数式・データ入力規則が含まれる。
openpyxl等で書き込むと**一部が壊れる/失われる恐れ**がある。そのため：

1. **原本を直接上書きしない**。必ず `*_backup_YYYYMMDD.xlsx` を残し、コピーで作業する。
2. スクリプトで書くのは **手入力4項目（A3/M2/C列/I列/K列）だけ**。数式セルには一切触れない。
3. 書き込み後は **必ずExcelで開いて**、①単価が全行入っているか ②合計が合うか
   ③レイアウト崩れ・画像消失が無いか を目視確認する。1件でもおかしければ手作業に切り替える。
4. どうしても不安な回は、スクリプトは**「貼り付け用ブロックの生成＋検証」までに留め**、
   実際のセル入力は人（またはExcel上でのコピペ）に任せる。これでも十分に時短になる。
5. 金額・取引先・請求月は **最後に必ず人＝山本社長が確認**（承認ルール）。

---

## 新しい取引先・新しい月を始める手順

1. **価格版を確認**：その取引先が使っている版（据置／▲10％／▼10％）を既存ファイルから踏襲。
2. **月次ファイルを用意**：`<取引先>様　202X年X月分　納品書・請求書.xlsx`（前月をコピー→中身クリア、
   またはマスタ版をコピー）。取引先名は `取引先リスト` の正式表記に合わせる。
3. **注文が来るたびに納品書N（日付別）へ追記**：手順1〜3を回す。
4. **月末に請求書を確認**して手順4〜5。
5. **正直に報告**：未マッチ・要確認点を明記して社長へ。承認後 `請求済み/` へ移動＋push。
