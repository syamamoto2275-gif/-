# 納品書・請求書フォーマット 構造メモ（実物解析）

対象：`Dropbox/ペット事業/販売/納品書・請求書フォーマット/SEED様_納品書フォーマット_1.5【据置】.xlsx`
（取引先ごとの月次ファイル `○○様 202X年X月分 納品書・請求書.xlsx` も同じ構造）

## シート一覧

| シート | 役割 |
|---|---|
| ご案内1〜4 | 送り状・案内文（固定文面） |
| 取引先リスト | 卸先名の正式表記一覧（表記ゆれの正解） |
| **データリスト** | 商品マスタ。単価はここから引く。**ここに無い商品名は単価が出ない** |
| **納品書1〜10** | 同一取引先の**日付別の納品**。1シート＝1回の納品 |
| **請求書** | 納品書1〜10を自動集計した月次請求 |
| 領収書 | 領収書フォーマット |
| 不足案内 | 欠品・不足の案内 |

> 重要：**1ワークブック＝1取引先＝1か月**。納品書N は「別の取引先」ではなく「同じ取引先の別の納品日」。

---

## データリスト（商品マスタ）

先頭行がヘッダ。商品は3行目以降。

| 列 | 内容 | 例 |
|---|---|---|
| A | **商品名（自動）** `=C&"　"&D&"　"&E`＝`商品名　カラー　サイズ` | `陶器ハウス　ピンク　`（末尾に全角スペースが残る場合あり） |
| B | 卸価格№ | `1` |
| C | 商品名 | `陶器ハウス` |
| D | カラー | `ピンク` |
| E | サイズ | （空のことも） |
| F | JAN | `4573298440118` |
| G | バーコード | `X5H32JIKPPLMMTZ` |
| H | **価格（卸単価）** | `2250` |

- 納品書のVLOOKUPは **データリスト A〜J 範囲** を検索キー=A列（正式名称）で引く：
  - `col 2` → B＝卸価格№
  - `col 6` → F＝JAN
  - `col 8` → H＝単価
- つまり **納品書C列の文字列 = データリストA列の文字列** が完全一致して初めて単価が出る。
- A列は数式なので、機械照合するときは **C＋"　"＋D＋"　"＋E を自分で組み立てて比較**すると確実
  （キャッシュ値に頼らない）。

## 納品書N シート（入力シート）

| セル | 内容 | 手入力? |
|---|---|---|
| `A1` | 「 納品書 」タイトル | 固定 |
| `A3` | **取引先名** | ✅ 手入力（`H3`＝「様」は固定） |
| `M2`(=`L2`ラベル「発行日」) | **発行日＝その納品日** | ✅ 手入力 |
| `J5〜J9` | 自社情報（shop fuu / 株式会社SEED 住所・TEL・事業者番号 T5180001078443） | 固定 |
| `Q3` | 検索用キーワード（候補絞り込み補助。任意） | 任意 |
| 11行目 | 見出し：№ / 商品名 / JAN / 発注番号 / 単価 / 数量 / 金額 / 備考 | 固定 |
| `C12`〜 | **商品名（データリストA列と完全一致）** | ✅ 手入力 |
| `I12`〜 | **発注番号**（例 `20260701－1`） | ✅ 手入力 |
| `K12`〜 | **数量** | ✅ 手入力 |
| `A12`〜 | 行番号 `=SUBTOTAL(103,...)` | 自動 |
| `B12`〜 | 卸価格№ `VLOOKUP(C, データリスト, 2)` | 自動 |
| `H12`〜 | JAN `VLOOKUP(C, データリスト, 6)` | 自動 |
| `J12`〜 | 単価 `VLOOKUP(C, データリスト, 8)` | 自動 |
| `L12`〜 | 金額 `=IFERROR(J*K,"")` | 自動 |
| `M12`〜 | 備考（例「不足2」） | 任意・手入力 |

- データ行は **12行目から** 開始（40行目付近まで枠がある）。
- `S`〜`T`列には検索用の作業数式（`データリスト`の絞り込み結果）が入る。**触らない**。
- **単価（J列）が空欄の行＝商品名マッチ失敗**。転記後はここを必ずチェックする。

## 請求書 シート（自動集計）

| セル | 内容 |
|---|---|
| `A1` | 「 請求書 」タイトル |
| `A3` | 取引先名 `=納品書1!A3`（自動） |
| `K2` | 「発行日」ラベル |
| `B11` | **請求月**（例「2026年12月」）＋ `E11`「納品分」 |
| `B12` | 「ご請求合計金額」 / `F12` = `=K420`（合計セル）/ `I12`「円」 |
| 14行目 | 見出し：№ / 納品日 / 商品名 / JAN / 発注番号 / 単価 / 数量 / 金額 |
| `C15`〜 | 各行が `=納品書1!Bxx` `=納品書1!Cxx` …で **納品書の内容を引き写す** |
| `N`列 | 参照元シート名（`納品書1` など）を記録 |

- 15行目以降が **納品書1 → 納品書2 → …** の順に各納品書の明細を積み上げて集計している。
- `ご請求合計金額` は末尾の合計行（`K420`付近）を参照。**金額は全部自動**なので、
  人がやるのは「納品書N を正しく埋める」ことだけ。
- 確認ポイント：①請求月（B11）②取引先（A3）③合計（F12）が各納品書合計と一致。

## よくある失敗と対策

| 失敗 | 原因 | 対策 |
|---|---|---|
| 単価が空欄 | C列の商品名がデータリストA列と不一致（スペース・カラー・サイズ違い） | `match_products.py`で照合。全角スペース・末尾スペースに注意 |
| 合計が合わない | 数量未入力／マッチ漏れ行がある | 単価空欄行をゼロにする前に必ず確認 |
| レイアウト/画像が消えた | スクリプトでワークブックを保存し直した | 原本コピーで作業＋保存後にExcelで目視確認（SKILL.md 安全ルール） |
| 違う価格で請求 | 価格版（据置/▲10%/▼10%）を取り違え | 取引先ごとの既存ファイルの版を踏襲。迷ったら社長確認 |
