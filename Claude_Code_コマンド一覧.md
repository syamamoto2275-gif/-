# Claude Code コマンド一覧

作成日: 2026-06-18

---

## スラッシュコマンド（`/` で始まるコマンド）

| コマンド | 説明 |
|---|---|
| `/help` | ヘルプを表示する |
| `/clear` | 会話の履歴をクリアする（新しい会話を始める） |
| `/compact` | 会話を要約して圧縮する（長くなったときに使う） |
| `/config` | 設定を変更する（テーマ・モデルなど） |
| `/permissions` | ツールの実行許可を管理する |
| `/hooks` | フック（自動実行スクリプト）を設定する |
| `/agents` | エージェントの一覧を確認する |
| `/doctor` | 環境の診断を行う |
| `/fast` | Fast モード（高速・高性能）のON/OFF切り替え |
| `/memory` | メモリ（記憶）の内容を確認・編集する |
| `/init` | プロジェクトの CLAUDE.md を自動生成する |
| `/review` | プルリクエストのコードレビューを行う |
| `/code-review` | 現在の差分をレビューする（例: `/code-review high`） |
| `/code-review ultra` | クラウドで深い多エージェントレビューを行う |
| `/run` | アプリを起動して動作確認する |
| `/verify` | コード変更が正しく動いているか確認する |
| `/loop` | コマンドを繰り返し実行する（例: `/loop 5m /review`） |
| `/schedule` | 定期実行タスクを設定する |
| `/security-review` | セキュリティレビューを行う |

---

## よく使うキーボードショートカット

| キー操作 | 説明 |
|---|---|
| `Enter` | メッセージを送信する |
| `Shift + Enter` | 改行する（送信しない） |
| `↑` キー | 直前に入力したメッセージを呼び出す |
| `Ctrl + C` | 実行中の処理を中断する |
| `Ctrl + L` | 画面をクリアする（`/clear` と同じ） |
| `Escape` | 入力をキャンセルする |

---

## コード レビューの詳細オプション

`/code-review` にはレベルを指定できます。

| オプション | 意味 |
|---|---|
| `/code-review low` | 簡易レビュー（高信頼度の重大な問題のみ） |
| `/code-review medium` | 標準レビュー |
| `/code-review high` | 詳細レビュー（広範囲をチェック） |
| `/code-review ultra` | クラウドで最も深いレビュー（有料・時間かかる） |
| `/code-review --fix` | レビュー結果を自動で修正も行う |
| `/code-review --comment` | GitHub PR にコメントとして投稿する |

## エージェント（Agent）の種類

Claude Code が内部で使う専門エージェントです。

| エージェント名 | 役割 |
|---|---|
| `claude` | 汎用エージェント（何でも対応） |
| `Explore` | コードの検索・場所の特定に特化 |
| `Plan` | 実装計画の設計に特化 |
| `claude-code-guide` | Claude Code 自体の使い方の質問に特化 |

---

## メモリの種類

Claude が記憶として保存できる情報の分類です。

| 種類 | 内容 |
|---|---|
| `user` | ユーザーの役割・好み・スキルレベル |
| `feedback` | 作業の進め方に関する指示・フィードバック |
| `project` | 進行中の作業・目標・期限 |
| `reference` | 外部システム・URLなどへのポインタ |

---

## よく使うファイル

| ファイル | 役割 |
|---|---|
| `CLAUDE.md` | プロジェクトのルール・指示書（Claude が自動で読む） |
| `.claude/settings.json` | プロジェクトごとの設定（権限・フックなど） |
| `~/.claude/settings.json` | 全プロジェクト共通の設定 |
| `~/.claude/projects/.../memory/` | 自動メモリの保存場所 |

---

## 使い方のヒント

- **わからないことはそのまま日本語で聞いてOK**。Claude が適切なコマンドを判断して使います。
- `/clear` を使うと会話がリセットされ、新鮮な状態で作業を再開できます。
- 長い会話は `/compact` で圧縮すると、応答が速くなることがあります。
- `Ctrl + C` はいつでも使えます。処理を止めたいときにすぐ押してください。
