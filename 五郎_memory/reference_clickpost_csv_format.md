クリックポスト（日本郵便）の「まとめ入力用CSV」は**見出し行あり・固定8列**。1列でも足りないと「1行目の項目数が足りません／先頭8項目までを変更してはいけません」で弾かれる。

**正しい列順（8列）：**
1. お届け先郵便番号（ハイフンあり/なしどちらも可の半角数字）
2. お届け先氏名
3. お届け先敬称（例：様）
4. お届け先住所1行目
5. お届け先住所2行目
6. お届け先住所3行目
7. **お届け先住所4行目（未使用でも空欄で列は必ず置く）**
8. **内容品（必ず8列目）**

**やりがちな罠：** 内容品を7列目に置き、住所4行目の列を省くと7列になり拒否される（2026-07-01 ペライチ大改修で発生→07-02 修正）。出荷マクロの「クリック一括発行アップデータ」は必ずこの8列で出力し、見出し行も実行のたびに書き直すこと。関連：[[project_vba_shipping]] のペライチ／[[reference_excel_numeric_address_date_trap]]（郵便・住所は@書式で日付化防止）。
