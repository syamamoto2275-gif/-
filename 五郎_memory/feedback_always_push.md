**ルール0（最重要・例外なし）：ファイル・記憶・社長から得た情報を作成/変更したら、その場で即座にGitHubリポジトリへ commit → push する。「同期して」と言われるのを待たない。**

**ルール0-2（社長の明示指示 2026-06-26）：その場その場で、記憶を「すべて」即アップデートする。** 新しい事実・訂正を社長から聞いたら、関係するファイルを1つだけでなく**全部**その場で直す（記憶ファイル＋CLAUDE.md＋設計図＋組織図＋共通ルールなど、矛盾が残らないように横断的に更新）→ 即push。「あとで直す」「片方だけ直す」は禁止。情報の不整合を1秒たりとも残さない。

**共通の置き場（2台の五郎の唯一の同期手段）：**
- GitHubリポジトリ `syamamoto2275-gif/-`（[[reference_github_repo]]）／ローカル: `C:\Users\山本　史朗\OneDrive\Desktop\新しいフォルダー\-`
- 共通ルール本体は repo内 `五郎_共通ルール.md`。会話ログは `五郎_会話ログ_YYYY-MM-DD.md`。
- `.claude/memory` はPCごとに独立で**同期されない**。だから大事なことは必ずrepo内のファイルに書き出してpushする。

**運用：**
- 作業開始時：`git pull` で最新を取得し、もう一方のPCの五郎が残したログ・ルールを読んでから入る。
- 作業中／作業後：成果物・決定事項・社長の指示・人物情報を `五郎_会話ログ_日付.md` に書き、即 add→commit→push。
- 言葉遣い：社長・お客様・目上には「了解」を使わない。「承知しました」等を使う。

**Why:** 即pushしないと、翌日もう片方のPCの五郎が把握できず、社長に同じ説明を二度させ、社長の時間を無駄にする。2026-06-26に自宅PCの五郎（私）が落合さんの情報・手順書を作ったのにpushせず、実際にこの失敗を起こした。**二度と繰り返さない。**
